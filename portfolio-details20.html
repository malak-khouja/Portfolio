Building RAG Chatbots for Technical Documentation

This project demonstrates the implementation of a Retrieval Augmented Generation (RAG) chatbot designed to provide context-aware answers about car warning messages from technical manuals. Developed as a proof of concept for a car manufacturer, the system integrates Large Language Models (LLMs) with automotive technical documentation to create an intelligent assistant that could eventually be integrated with text-to-speech software for in-vehicle use.

Key Features:

  • Document Processing: Loads and parses HTML car manual documentation (MG ZS warning messages) using UnstructuredHTMLLoader, extracting structured information about warning messages, meanings, and recommended actions.
  • Intelligent Text Splitting: Implements RecursiveCharacterTextSplitter with 1000-character chunks and 200-character overlap to maintain context while optimizing retrieval performance.
  • Vector Database Storage: Utilizes ChromaDB for efficient storage and semantic search of document embeddings, enabling fast and accurate retrieval of relevant context.
  • Semantic Search: Converts user queries and document chunks into vector embeddings using OpenAI's text-embedding-3-small model for similarity-based retrieval.
  • Context-Aware Responses: Uses GPT-4o-mini to generate accurate, concise answers (max 3 sentences) based only on retrieved context from the car manual.
  • RAG Pipeline: Implements a complete LangChain pipeline that chains retrieval, prompt formatting, and language model generation for end-to-end question answering.

Technologies Used:

  • Programming Language: Python for all data processing, model integration, and pipeline orchestration.
  • LLM Framework: LangChain (langchain-core 0.3.72) for building the RAG pipeline with modular components.
  • Language Model: OpenAI GPT-4o-mini for natural language generation with temperature=0 for consistent outputs.
  • Embeddings: OpenAI text-embedding-3-small for converting text to semantic vectors.
  • Vector Database: ChromaDB (langchain-chroma 0.2.5) for efficient storage and similarity search.
  • Document Processing: Unstructured 0.18.11 for parsing HTML technical documentation.
  • Development Environment: Jupyter Notebook for interactive development and testing.

Technical Implementation:

  • Retrieval Strategy: Similarity-based search to find the most relevant document chunks for each query.
  • Prompt Engineering: Carefully designed prompts that instruct the LLM to answer only from provided context and admit when information is unavailable.
  • Grounded Generation: Prevents hallucinations by restricting answers to retrieved manual content.
  • Scalable Architecture: Modular design allows easy extension to additional vehicle models and documentation.

Project Impact: This project provides a scalable, production-ready framework for integrating technical documentation with LLMs. It demonstrates expertise in modern NLP techniques, RAG architecture, and practical AI application development. The system enhances driver safety by providing instant, accurate guidance on vehicle warning messages and could be extended to voice interfaces for hands-free operation.
